# RB-005.1 Exfiltration Alert Triage

## Objective

Validate and triage suspected data exfiltration alerts to determine legitimacy, severity, exposure risk, and immediate containment requirements.

---

## Status

Live

---

## Inputs

- DLP alerts
- Proxy logs
- Firewall telemetry
- Cloud audit logs
- Endpoint telemetry
- User reports

---

## Tools

- DLP platform
- SIEM
- Proxy/firewall platform
- Cloud audit tooling
- EDR platform

---

## Steps

### 1. Validate Alert Authenticity

Determine:
- Alert source
- Detection logic
- Alert confidence
- Trigger conditions

Review:
- Alert metadata
- Transfer characteristics
- User context
- Data classification indicators

---

### 2. Identify Impacted Assets

Collect:
- User identity
- Endpoint hostname
- Source IP address
- Destination infrastructure
- SaaS platform involvement

---

### 3. Assess Transfer Characteristics

Review:
- Transfer volume
- File types
- Transfer timing
- Encryption usage
- Transfer protocol

---

### 4. Review User Context

Determine:
- Business justification
- Historical transfer behaviour
- Administrative access
- Recent suspicious activity

---

### 5. Assess Immediate Risk

Determine:
- Is exfiltration ongoing?
- Is regulated data involved?
- Is sensitive intellectual property involved?
- Is external attacker activity suspected?

---

## Decision Points

| Condition | Action |
|----------|--------|
| False positive suspected | Validate before escalation |
| Active exfiltration ongoing | Immediate containment |
| Regulated data involved | Escalate to legal/compliance |
| Multiple systems involved | Expand investigation scope |

---

## Output

- Alert validation status
- Initial exposure assessment
- Impacted asset inventory
- Recommended containment actions

---

## Common Failure Modes

- Treating large transfers as malicious without context
- Ignoring encrypted transfer channels
- Missing SaaS-based transfers
