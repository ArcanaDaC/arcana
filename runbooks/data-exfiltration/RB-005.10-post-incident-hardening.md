# RB-005.10 Post-Incident Hardening

## Objective

Improve security controls, detection coverage, governance, and operational resilience following a data exfiltration incident.

---

## Status

Live

---

## Inputs

- Incident timeline
- Exposure findings
- Root cause analysis
- Detection gaps
- Recovery findings

---

## Tools

- DLP platform
- SIEM
- CASB platform
- Identity provider
- Security governance tooling

---

## Steps

### 1. Review Root Cause

Identify:
- Initial access vector
- Control failures
- Monitoring gaps
- Governance failures
- Detection weaknesses

---

### 2. Improve DLP Coverage

Enhance:
- Sensitive data monitoring
- File movement visibility
- SaaS transfer monitoring
- Endpoint DLP controls
- Cloud storage visibility

---

### 3. Harden SaaS & Cloud Controls

Review and improve:
- Sharing restrictions
- OAuth governance
- External collaboration policies
- CASB enforcement
- Tenant visibility

---

### 4. Improve Identity Security

Enhance:
- MFA enforcement
- Conditional access policies
- Session lifetime controls
- Privileged access restrictions
- Risk-based authentication

---

### 5. Improve Detection & Response

Develop:
- New detections
- Threat hunting queries
- Behavioural analytics
- Transfer anomaly detection
- Insider risk detections

---

### 6. Conduct Lessons Learned Review

Document:
- Response successes
- Operational gaps
- Escalation issues
- Tooling limitations
- Process improvements

---

## Decision Points

| Condition | Action |
|----------|--------|
| Detection gaps identified | Prioritise engineering improvements |
| Governance failures identified | Escalate policy review |
| Repeated control failures identified | Executive escalation |

---

## Output

- Hardening recommendations
- Detection improvement backlog
- Governance improvement actions
- Lessons learned summary

---

## Common Failure Modes

- Treating recovery as closure
- Ignoring governance failures
- Failing to operationalise lessons learned

---

## Automation Hooks

- Auto-detection deployment
- Auto-governance validation
- Auto-policy compliance checks
- Auto-risk scoring improvements
