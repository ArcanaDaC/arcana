# RB-005.2 Outbound Traffic Analysis

## Objective

Analyse outbound network activity to identify exfiltration channels, transfer methods, and external infrastructure involved in data theft activity.

---

## Status

Live

---

## Inputs

- Proxy logs
- Firewall telemetry
- NetFlow data
- DNS telemetry
- Endpoint network telemetry

---

## Tools

- SIEM
- Firewall platform
- Proxy platform
- NetFlow analysis tools
- Threat intelligence platform

---

## Steps

### 1. Identify Outbound Destinations

Review:
- External IP addresses
- Domains
- Cloud storage destinations
- SaaS platforms
- Anonymisation infrastructure

---

### 2. Assess Transfer Volume & Timing

Determine:
- Transfer size
- Duration
- Frequency
- Burst behaviour
- Time-of-day anomalies

---

### 3. Identify Transfer Protocols

Review:
- HTTPS
- FTP/SFTP
- WebDAV
- SMB over VPN
- API-based transfers

---

### 4. Assess Encryption Usage

Identify:
- Encrypted channels
- Custom encryption
- Archive encryption
- TLS anomalies

---

### 5. Correlate Endpoint Activity

Review:
- Associated processes
- User context
- File access patterns
- Browser activity
- Cloud sync activity

---

### 6. Assess Ongoing Activity

Determine:
- Is exfiltration still active?
- Are additional systems involved?
- Is automated transfer behaviour present?

---

## Decision Points

| Condition | Action |
|----------|--------|
| Active transfer detected | Immediate containment |
| Suspicious cloud storage identified | Proceed to RB-005.4 |
| Multiple systems involved | Expand scope investigation |

---

## Output

- Outbound transfer inventory
- Destination infrastructure assessment
- Transfer behaviour analysis
- Active exfiltration status

---

## Common Failure Modes

- Focusing only on known malicious infrastructure
- Ignoring legitimate SaaS abuse
- Missing low-and-slow transfers

---

## Automation Hooks

- Auto-transfer anomaly detection
- Auto-destination enrichment
- Auto-flow correlation
- Auto-risk scoring
