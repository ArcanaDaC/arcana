# RB-005.4 Cloud Storage & SaaS Review

## Objective

Identify unauthorised cloud storage usage, SaaS abuse, public exposure, and cloud-based exfiltration activity.

---

## Status

Live

---

## Inputs

- Cloud audit logs
- SaaS telemetry
- OAuth logs
- File sharing telemetry
- Authentication logs

---

## Tools

- CASB platform
- Cloud audit tooling
- SaaS administration consoles
- SIEM
- Identity provider

---

## Steps

### 1. Identify Cloud Storage Usage

Review:
- Dropbox
- Google Drive
- OneDrive
- Box
- Mega
- Other file sharing platforms

---

### 2. Review File Sharing Activity

Assess:
- Public link creation
- External sharing
- Anonymous access
- Guest access
- Link permission changes

---

### 3. Assess OAuth & API Usage

Identify:
- Suspicious third-party integrations
- Excessive API usage
- Rogue OAuth applications
- Persistent delegated access

---

### 4. Review SaaS Upload Activity

Determine:
- Upload volumes
- Sensitive file uploads
- Transfer timing
- User behaviour anomalies
- Shared external destinations

---

### 5. Assess Public Exposure

Identify:
- Publicly accessible files
- Misconfigured sharing
- Exposed repositories
- Anonymous download capability

---

### 6. Validate Tenant Integrity

Confirm:
- No malicious integrations remain
- Sharing controls operational
- Excessive permissions removed
- Monitoring enabled

---

## Decision Points

| Condition | Action |
|----------|--------|
| Public exposure identified | Immediate remediation |
| Tenant-wide SaaS abuse identified | Expand scope investigation |
| OAuth persistence identified | Escalate to PB-004 |

---

## Output

- SaaS exposure assessment
- Cloud storage activity timeline
- Public exposure findings
- OAuth abuse assessment

---

## Common Failure Modes

- Ignoring SaaS-native sharing
- Missing OAuth persistence
- Focusing only on traditional network exfiltration

---

## Automation Hooks

- Auto-sharing visibility checks
- Auto-OAuth anomaly detection
- Auto-public exposure discovery
- Auto-upload anomaly monitoring
