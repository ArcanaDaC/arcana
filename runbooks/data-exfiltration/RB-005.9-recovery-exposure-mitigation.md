# RB-005.9 Recovery & Exposure Mitigation

## Objective

Reduce ongoing exposure risk, restore secure operations, and mitigate follow-on abuse following a data exfiltration incident.

---

## Status

Live

---

## Inputs

- Exposure assessment
- Containment status
- Authentication telemetry
- Endpoint investigation findings
- Cloud audit findings

---

## Tools

- Identity provider
- EDR platform
- DLP platform
- CASB platform
- SIEM

---

## Steps

### 1. Validate Containment Completion

Confirm:
- Exfiltration activity stopped
- Sessions revoked
- Tokens invalidated
- Endpoints isolated or rebuilt
- Malicious integrations removed

---

### 2. Remove Exposure Paths

Review and remediate:
- Public file sharing
- SaaS misconfigurations
- OAuth abuse
- Unapproved integrations
- Excessive permissions

---

### 3. Reset Compromised Access

Perform:
- Password resets
- MFA resets
- API key rotation
- Service account credential rotation
- Certificate replacement where required

---

### 4. Restore Secure Operations

Validate:
- Secure access controls
- Approved sharing configurations
- Monitoring functionality
- Data access restrictions
- Endpoint integrity

---

### 5. Monitor for Follow-On Abuse

Review:
- Repeated transfer attempts
- Reuse of attacker infrastructure
- Credential reuse attempts
- Suspicious SaaS access
- Additional staging behaviour

---

### 6. Coordinate Business Recovery

Support:
- Business owner validation
- User recovery
- Service restoration
- Stakeholder updates
- Operational continuity

---

## Decision Points

| Condition | Action |
|----------|--------|
| Continued suspicious activity observed | Reassess containment |
| Additional exposure identified | Expand incident scope |
| Persistent SaaS abuse identified | Escalate investigation |

---

## Output

- Recovery status
- Exposure mitigation status
- Credential rotation status
- Service restoration timeline

---

## Common Failure Modes

- Restoring access before exposure paths removed
- Ignoring long-lived API credentials
- Failing to monitor for follow-on abuse

---

## Automation Hooks

- Auto-credential rotation
- Auto-sharing policy validation
- Auto-follow-on activity monitoring
- Auto-access review enforcement
