# RB-002.2 Identify Patient Zero

## Objective

Identify the earliest known compromised system and determine the likely initial access vector.

---

## Inputs

- EDR telemetry
- Authentication logs
- SIEM timeline
- Endpoint timelines
- Ransomware execution timestamps

---

## Tools

- SIEM
- EDR platform
- Active Directory logs
- Email telemetry
- VPN logs

---

## Steps

### 1. Build Initial Timeline

Determine:
- Earliest encryption event
- First ransomware execution
- First suspicious authentication event

---

### 2. Identify Earliest Impacted Host

Review:
- File modification timestamps
- EDR detections
- User reports
- Network telemetry

---

### 3. Review Initial Access Indicators

Check for:
- Phishing activity
- VPN compromise
- Exposed RDP
- Credential theft
- Malware download activity

---

### 4. Analyse User Activity

Review:
- Privileged account usage
- Unusual login locations
- Service account activity
- MFA anomalies

---

### 5. Identify Propagation Path

Determine:
- Lateral movement techniques
- Shared credential usage
- Administrative tool usage
- Remote execution methods

---

## Decision Points

| Condition | Action |
|----------|--------|
| Phishing identified | Escalate to PB-001 |
| Privileged account abuse | Escalate to RB-002.6 |
| Internet-facing exposure identified | Emergency exposure review |

---

## Output

- Suspected patient zero
- Initial access vector
- Timeline of compromise
- Propagation path summary

---

## Common Failure Modes

- Assuming first detected host is patient zero
- Ignoring authentication telemetry
- Missing dormant attacker activity prior to encryption

---

## Automation Hooks

- Auto-build event timeline
- Auto-correlate authentication events
- Auto-identify earliest detections
