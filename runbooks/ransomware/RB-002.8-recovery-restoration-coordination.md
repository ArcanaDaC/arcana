# RB-002.8 Recovery & Restoration Coordination

## Objective

Coordinate safe and controlled restoration of systems and services following ransomware containment.

---

## Inputs

- Recovery prioritisation list
- Backup validation status
- Containment status
- Infrastructure dependency mapping
- Business impact assessment

---

## Tools

- Backup platform
- Virtualisation platform
- Asset inventory
- SIEM
- EDR platform

---

## Steps

### 1. Validate Containment Status

Confirm:
- Propagation stopped
- Persistence mechanisms addressed
- Privileged access reviewed
- Compromised systems isolated

---

### 2. Prioritise Recovery Activities

Restore based on:
- Business criticality
- Dependency order
- Infrastructure requirements
- Recovery feasibility

---

### 3. Restore Systems in Phases

Use staged recovery:
- Core infrastructure
- Identity systems
- Critical applications
- User endpoints

---

### 4. Validate Restored Systems

Check:
- System functionality
- Malware absence
- Authentication integrity
- Network connectivity

---

### 5. Monitor for Reinfection

Monitor:
- EDR telemetry
- Authentication anomalies
- File modification spikes
- Suspicious outbound traffic

---

### 6. Reintroduce Systems Carefully

Gradually:
- Reconnect segmented systems
- Restore trust relationships
- Re-enable network pathways

---

## Decision Points

| Condition | Action |
|----------|--------|
| Reinfection activity observed | Halt restoration immediately |
| Authentication instability detected | Reassess identity integrity |
| Backup corruption discovered | Pause restoration planning |

---

## Output

- Recovery status
- Restored system inventory
- Reinfection monitoring status
- Business restoration progress

---

## Common Failure Modes

- Restoring too quickly
- Reconnecting infected systems
- Ignoring dependency order
- Failing to monitor for reinfection

---

## Automation Hooks

- Auto-monitor restoration health
- Auto-alert on reinfection indicators
- Auto-track restoration phases
